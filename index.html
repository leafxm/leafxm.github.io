一步步教你用GitHub Pages与Hexo搭建静态博客

简介 记录windows环境下一步步搭建博客的过程~因为自己在搭博客过程中遇到一些小问题，所以提醒别人不要踩坑（内心os：喂，你以为人人都和你一样二吗），同时也是对这个过程的一个回顾与思考，作为第一篇博文再合适不过。力求条理清楚，内容主要分为GitHub设置、Hexo设置、利用Hexo上传文件到GitHub和域名绑定这四部分。

软件安装

Git 下载安装

首先，下载 Git.
安装大部分选择默认，出于操作方便考虑，PATH选择如图：

原因如下：

Git的默认设置下，出于安全考虑，只有在Git Bash中才能进行Git的相关操作。按照上图进行的选择，将会使得Git安装程序在系统PATH中加入Git的相关路径，使得你可以在CMD界面下调用Git，不用打开Git Bash了。

接下来，检查Git安装正确没有，打开cmd，输入

git --version`

如果结果显示的是git version 2.7.0.windows.1(具体版本和下载的一致)则安装成功。
打开Git可以通过两种方式：

  • 在开始菜单-所有程序-Git文件夹找到 “Git Bash”，点击打开；
  • 在相应文件夹鼠标右键点击，可以点击
    “Git Bash Here” 选项来打开。

node.js 下载安装

下载 node.js，一路默认设置安装上就好了。然后检查是否安装正确，打开cmd，输入

node -v 
npm -v

如果正确显示版本号则安装正确。

GitHub设置

GitHub 账号注册

首先是注册GitHub账号。如果你已经有了一个GitHub账号，那么可以愉快跳过这小段啦。打开Github首页，输入用户名、email和密码。

tips ：开始博主是用126邮箱无论如何收不到验证邮件，后来换qq邮箱就秒收到啦。邮件验证很重要，关系到创建项目。所以一定要验证哦！换邮箱还给博主带来了意想不到的麻烦，下面你就会看到博主的辛酸经历了~

GitHub 账号设置

接下来登录账号，点击页面右上角的”+”号，选择”New repository”。如图所示：

进入创建页面，在Repository name 中填写”yourname.github.io”, Description 写一些介绍，也可以空着。

接下来需要开启gh-pages功能。

如上图选择settings，在页面找到GitHub Pages，点击Automatic page generator，Github会自动创建出一个gh-pages的页面。如果配置没有问题，稍等片刻yourname.github.io这个网址就可以正常访问了，也意味着
GitHub设置部分结束啦。

Hexo设置

Hexo 安装和初始化

接下来，安装Hexo。选择一个地方创建一个文件夹(下面以文件夹名“hexo文件夹”指代该文件夹)，在hexo文件夹空白处按Shift+鼠标右键，点击“在此处打开命令行窗口”（在该文件夹打开命令行的方式）。命令行中输入

npm install hexo-cli -g
npm install hexo --save

接下来我们看一下Hexo安装好了没有，命令行输入

hexo -v

若看到如下内容则安装成功：

Hexo初始化，在命令行输入hexo init，然后输入npm install，npm会自动安装啦~

本地预览博客

输入hexo g和hexo s即可在http://localhost:4000/预览博客啦。按 Ctrl+C 之后，预览效果就会消失~所以每次预览前都要用那两个命令哦~
至此，简单的Hexo设置部分就结束啦。接下来，就是如何将Hexo本地内容上传到GitHub上。

利用Hexo上传文件到GitHub

配置SSH keys

为了让本地文件和远程的GitHub联系，可以使用SSH keys。

首先，生成新的SSH key。在hexo文件夹处，点击鼠标右键，选择“Git Bash Here”，在git bash窗口输入
$ ssh-keygen -t rsa -C “mailname@mail.com”
“mailname@mail.com”是你的邮箱地址，出现一段文字，按回车，然后会提示输入密码，这个密码是提交项目时使用的，主要是为了防止别人往你的项目里提交内容（不会显示*号的，继续写就好），也可以空着就表示没有密码。（不要忘记这个密码哦，以后每次往上传文件到GitHub都要输入的~）如果看到”The key’s randomart image is”一个奇奇怪怪的图形，那么这一步就成功啦！上面会有说”Your public key have been saved in…”,记住这个地址哦。

打开上面那个地址，打开 id_rsa.pub 文件，复制内容。登录GitHub，在项目页进入settings（第一部分有截图标注位置），选择最后一个”Deploy keys”,然后在右边找到”Add deploy key”点击,标题随便写，内容粘贴刚刚复制的id_rsa.pub文件里的内容（要准确才行），可写性选择的选项一定要打勾啊（忘记打勾而浪费不少时间的博主哭着提醒）！Add key 就ok啦。

接下来测试是否成功，在git bash 输入
$ ssh -T git@github.com
“git@github.com”不用修改，如果让你输入yes/no,输入yes就好，然后就会看到” You’ve successfully authenticated, but GitHub does not provide shell access.”这句表示成功的话了~

现在已经可以通过SSH链接到GitHub了，但是还要完善你的个人信息才可以。命令如下：

$ git config --global user.name "yourname"//用户名
$ git config --global user.email "yourname@mail.com"//填写自己的邮箱

这样这步就完成啦。

我之前按照的教程是最开始设置SSH key，所以是在126邮箱下，结果后来收不到验证邮件换了邮箱，，而SSH key就没有了我却不知道，导致花费了很多时间无法用hexo推送到github……（角落画圈圈中，直到写到这里才知道自己当初浪费那么多时间纠结那么久的原因）

Hexo 配置Deployment

打开hexo文件夹，找到_config.yml文件，打开这个文件，拉到最后，找到Deployment，然后按照如下修改：

deploy:
  type: git
  repo: git@github.com:yourname/yourname.github.io.    
  branch: master

这里的repo格式要和GitHub中的设置对应，打开你的项目页面，如下图选择SSH：

复制窗口里”git@github.com:…”到 repo。

在hexo文件夹右键打开git bash，输入

$ hexo g
$ hexo d

或者两个命令的合并

$ hexo d -g

现在你就可以打开你的git项目看文件是否改变了，打开你的页面yourname.github.io,看看你的页面是不是和hexo本地预览的效果一样了呢~如果是的，恭喜你成功啦~如果文件没有更新，你试着用以下命令：

$ hexo clean
$ hexo d -g

(我也不知道是因为等待时间不够问题还是别的原因，开始怎么也提交不上去，后来clean了一下就提交上去了……)不过还是建议看看命令运行后的反馈，如果有 fatal 这样的字，而自己不知道哪里出了问题，建议可以重新安装一下hexo试试。

tips：千万不要手贱改了_config.yml文件中的url,博主开始一直上传失败但不知道为什么（也许是SSH因为换了邮箱也清除了的原因）就破罐子破摔把url改成自己的域名了，之后hexo d和g就各种失败，而当时还不知道是这个原因，只能重装了hexo……后来想到hexo不好可能是因为改了url，测试了一下发现确实是的……

域名绑定

如果不想用自己的域名可以跳过这部分~

购买域名

博主是在godaddy上购买的，中文版网页，可以支付宝支付，很方便。注册账号（邮箱要用于验证，所以要填正确的邮箱地址哦）后，使用域名搜索来确定一个没有被购买的域名，选择后进入购物车，选择购买期限，填写结算信息，选择付款方式，付款成功后到邮箱查看激活邮件，激活后就可以在“我的账户-我的产品-域名”查看自己的域名了（没激活的时候是看不到的）~如果觉得说得比较简略，可以搜索一下详细教程哒~

DNS设置

因为godaddy被墙过有时候会无法解析，所以选择界面更好用更稳定免费的DNSpod来解析域名。
注册之后，点击”域名解析”添加域名，点击添加的域名，修改设置如下图：

前两条A的记录指向的ip地址是github Pages的提供的ip，www的记录是你在github注册的仓库，记录值是你的GitHub页面地址yourname.github.io。

接下来，要在godaddy网址修改DNS地址，将服务器改为DNSpod。登录goddady后，打开域名折叠框，在“动作”栏点击“管理DNS”，找到“域名服务器”栏，点击“管理”，如下图添加域名服务器就可以啦~

绑定独立域名与GitHub Pages的空间

登录GitHub，进入你建立的博客项目，在Repository的根目录下面，新建一个名为CNAME（注意不要写错啦，粗心的博主写错之后弄了半天没弄好才哭着发现问题是写错名字，而且是大写的哦）的文件，内容是你要绑定的域名，比如yourname.com。

注意啦，hexo文件夹中也要有这个文件哦，不然下次将hexo文件夹的内容上传到GitHub的时候，CNAME文件就会不见了，那域名就不管用啦。所以点击GitHub你建立的项目页的”Download Zip”，解压得到这个CNAME文件，复制到hexo文件夹中，就ok啦。

最后在github中打开自己的库，进入库的设置Settings界面，如果看到了提示“Your site is published at http://yourname.com.”，说明配置成功啦。

以上过程都需要一些时间，所以不要着急，稍等一段时间。如果发现有问题就利用搜索引擎查一下有没有别人也遇到类似问题，实在不行，可能是自己粗心哪一步做错了，可以重新来一遍。
