行内元素和块级元素的差别

块级元素就是默认display:block的元素，行内元素就是默认display:inline的元素，而通过display的设置可以让它们互相转换。

块级元素的特点：

  • 总是从新行开始
  • 宽度、高度、外边距以及内边距都可以控制。
  • 宽度默认是容器的100%，而与内容无关
  • 当没有定义height属性时，height其实和行高一样via）
  • 可以容纳内联元素和其他块元素。

常见的块级元素有：div、p、h1~h6、dl、dt、dd、form、ol、ul、li

内联元素的特点：

  • 和相邻行内元素在一行上
  • 设置高、宽无效，其宽度随着内容增加，高度随字号而改变（而不是行高）
  • 水平方向的padding和margin可以设置，垂直方向的无效。

常见的内联元素有：span a img inout select label em strong

position定位相关知识

position定位有四种属性：static（默认）、relative、absolute、fixed。除此之外还有float类型也可以用于定位。

  • static，没有定位，元素出现在文档流中。
  • relative，相对于其在文档流中的位置进行定位，没有脱离文档流。
  • absolute，元素脱离文档流，相对于相对于最近的已定位祖先元素（指有relative、absolute、fixed定位的元素），没有已定位祖先元素时相对于最初包含块定位。
  • fixed,与absolute只在于它是相对于视口定位。
  • float:left(right)让浮动框左右移动直到外边缘碰到包含框或另一个浮动框的边缘。浮动框不在文档普通流中。

一些性质：

关于定位的属性和叠加：

  • top、bottom、left、right的设置只在relative、absolute、fixed的情况下有效，在static以及只设置float的情况下无效。
  • 元素同时应用了position: relative、float、（top / left / bottom / right）属性后，则元素先浮动到相应的位置，然后再根据（top / left / bottom / right）所设置的距离来发生偏移
  • 元素同时应用了position: absolute及float属性，则float失效。
  • 第一个元素应用了position之后会覆盖着接下来的float元素（如果两个元素所处的位置相同）
  • 只有将float的元素的position设置成relative，才能通过设置float元素的z-index来的达到覆盖position:absolute的目的。
  • 对内联元素设置float和absolute属性，可以让元素脱离文档流，并且可以设置其宽高。但是float仍会占据位置，position会覆盖文档流中的其他元素。

行高和字号的继承

上面我们提到行高，关于行高还有一个小坑：line-height设置为150%和1.5这两种方式有什么区别？

首先，我们要知道当行高设置为比例（150%和1.5这种）形式时，它是根据字号的大小来计算出真正的行高px值。

每个元素都会按照它本身的字号乘以行高来设置自己的行高。当它没有字号只有行高的时候，会继承父元素的字号来计算行高。但是当它没有设置行高时，就要继承父元素的行高了，但是父元素行高为比例形式时，子元素的继承方式有差别。

当父元素的行高为150%时，会继承父元素计算出来的行高，也就是根据父元素的字体大小先计算出行高值然后再让子元素继承。所以当line-height:150%时，子元素的行高等于父元素字体大小 * 150%

而当父元素行高为1.5时，则是继承了行高值，再根据子元素的字体大小计算真正的行高。所以，当line-height:1.5时，子元素行高等于子元素字体大小 * 1.5。

那么关于字号又是如何继承的呢？当没有设置字号的时候，子元素会直接继承父元素的字号，当子元素字号设置为百分比的时候，将父元素的字号乘以百分比来计算。如果是em和rem呢？

em和rem

当em应用于font-size时，会根据父元素的大小来计算当前具体字号，而应用margin这样的属性的时候，则会根据当前元素的字号来计算。

所以当父元素字号为16px（可能也是根据它的父元素计算出来的具体值）时，子元素设置font-size: 2em; margin-bottom: 1em;此时它的字号是162=32px,而它的margin-bottom是321=32px。

为了解决这种混乱，rem就出现了。1rem等同于<html>中的font-size。当html的字号为16px时，上面那段代码，字号是32px（不论父元素字号是多少），而margin-bottom则是16px。

百分比

关于百分比是相对宽度高度这个那个计算很容易搞混，总结如下：

  • 乘以包含块的宽度： margin, padding, left, right, text-indent, width, max-width, min-width
  • 乘以包含块的高度： top, bottom, height, max-height, min-height
  • 乘以元素的字体大小：line-height
  • 乘以元素的行高：vertical-align
  • 背景定位中的百分比 background-position 分别设置水平方向和垂直方向上的两个值，如果使用百分比，那么百分比值会同时应用于元素和图像。例如 50% 50% 会把图片的（50%, 50%）这一点与框的（50%, 50%）处对齐，相当于设置了 center center。同理 0% 0% 相当于 left top，100% 100% 相当于 right bottom。
